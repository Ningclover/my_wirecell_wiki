# source-session-2026-04-25-pdhd-sp-deconvolution

Investigation of anomalous `summary_wiener` values on PDHD APA0 (V-plane ~31 ADC instead of ~200, W-plane stripe ch 1600–1759 low). Full read-through of the OmnibusSigProc deconvolution pipeline, ROI formation/refinement internals, and three-way config comparison (local / upstream apply-pointcloud / wcp-porting-validation).

## Confirmed findings

- `summary_wiener<N>_<ident>.npy` is the per-trace noise RMS from `ROI_formation::cal_RMS()` (not a literal "threshold" array, but the value used to compute all downstream thresholds) — see [[ROI Formation]]
- `m_nticks` is computed per-event from input frame tick span, not from config — see [[OmnibusSigProc]]
- `decon_2D_init()` performs: `load → FFT(time) → per-ch correction → FFT(wire) → ÷ response → wire filter → iFFT(wire) → iFFT(time) → wire shift → time shift → unpad → re-FFT(time)` — see [[OmnibusSigProc]]
- The per-wire RMS is computed once per wire per event in `find_ROI_by_decon_itself()` using a two-pass percentile + sigma-clipping estimator, before any ROI masking — see [[ROI Formation]]
- `r_th_factor` gates tight ROI admission (`load_data()`) and inter-wire connectivity pruning (`CheckROIs()`) — see [[ROI Refinement]]
- `r_fake_signal_low/high_th_ind_factor` scales the fake-signal cleanup thresholds for induction planes only (not collection W) in `CleanUpInductionROIs()` — see [[ROI Refinement]]
- Local PDHD config has `r_fake_signal_*_ind_factor = 0.15` for APA0 (vs 1.0 upstream), lowering induction thresholds 6.7×; upstream uses `r_th_factor = 2.5` for APA0 (vs 3.0 local) — see [[PDHD Signal Processing Configuration]]
- Local `params.jsonnet` uses `bf_test.json.bz2` as APA0 field response (vs official MCMC Garfield upstream) — see [[PDHD Signal Processing Configuration]]
- `FrameFileSink` saves `summary_<tag>_<ident>.npy` only if `frame->trace_summary(tag)` is non-empty; only wiener traces have a summary, gauss do not — see [[OmnibusSigProc]]
- `MaskSlices` reads `summary_wiener` and multiplies by `nthreshold[plane]=3.6` to form the per-tick, per-wire imaging cut — see [[PDHD Signal Processing Configuration]]
- `wcp-porting-validation/pdhd/` imports NF/SP configs from WCT install path; it does not carry its own copies → WCP-validation ≡ upstream for sp.jsonnet, nf.jsonnet, params.jsonnet, chndb-base.jsonnet, sp-filters.jsonnet — see [[PDHD Signal Processing Configuration]]
- Three-filter decon variant chain for induction: `decon_2D_tighterROI` → `decon_2D_tightROI` → `find_ROI_by_decon_itself(r_data, r_data_tight)` → `decon_2D_looseROI` → `find_ROI_loose` → `decon_2D_ROI_refine` — see [[OmnibusSigProc]]

## Context

User is processing PDHD run 027409 event 2 at `/exp/dune/data/users/xning/proto-dune-hd/SignalProcessing`. Observed that `summary_wiener0` (APA0) had V-plane values ~31 ADC (6× below normal ~200) and a W-plane stripe (ch 1600–1759) also anomalously low. The standalone NF+SP re-run produced correct values; the anomaly was traced to the `input_data` SP archives generated by Xuyang's art/LArSoft chain with a different config. Config comparison revealed the `_ind_factor=0.15` and `bf_test.json.bz2` discrepancies in the local pdhd configs vs upstream.
